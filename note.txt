I. LÝ THUYẾT (note lại)
- Java Spring (Spring) là một Framework mạnh mẽ của Java, dùng để xây dựng ứng dụng Web, API, Microservices hoặc hệ thống doanh nghiệp lớn
- Spring giúp viết code Java ít hơn, dễ bảo trì hơn, an toàn và mở rộng hơn
- Sử dụng Spring Boot: có thể hiểu là việc cấu hình dự án cho nó chạy nhanh hơn, nó là thành phần bên trong hệ sinh thái Java (Java Ecosystem)
- nhiều công ty và doanh nghiệp đang sử dụng công nghệ Java Spring (Spring Boot)

III. CÁC MODULE THƯỜNG GẶP:
- Spring Boot: Giúp khởi động ứng dụng nhanh, tự cấu hình
- Spring Core: Nền tảng gốc, dependency injection, quản lý đối tượng
- Spring Data: Làm việc với database (MySQL, SQL Server, PostgreSQL, ...) dễ dàng
- Spring Security: Bảo mật, đăng nhập, phân quyền, xác thực token (JWT, OAuth2)
- Spring Web / REST: Xây dựng website và API
- Spring Cloud: Dành cho hệ thống lớn, microservices

2. Template Engine
a. Thymeleaf (phổ biến nhất hiện tại cho Spring MVC)
- Cấu hình trong file pom.xml:
                                <dependency>
                                    <groupId>org.springframework.boot</groupId>
                                    <artifactId>spring-boot-starter-thymeleaf</artifactId>
                                </dependency>
- Đây gần như là “default choice” khi làm server-side rendering với Spring Boot.
- Mặc định, Thymeleaf sẽ tìm các file template (HTML) trong thư mục: src/main/resources/templates

b. JSP (legacy nhưng từng rất phổ biến)
- Ngày xưa đây là chuẩn gần như mặc định của Java web.

c. FreeMarker
- Một template engine lâu đời và khá mạnh.

d. Mustache
- Template engine cực tối giản.

e. Pebble
- Template engine hiện đại kiểu giống Jinja2/Twig.


3. Ioc (Inversion of Control) - Sự đảo ngược điều khiển
ĐỊNH NGHĨA:
- Là nguyên tắc thiết kế (Design Pattern) phần mềm, không phải là một công nghệ cụ thể.
- Nó là nền tảng cốt lõi của Spring Framework

BẢN CHẤT: 
- Thay đổi cách thức kiểm soát luồng hoạt động và quản lý đối tượng trong ứng dụng.

SO SÁNH:
- Cách truyền thống (Không có Ioc): Lập trình viên (code) phải chịu trách nhiệm hoàn toàn trong việc:
+ Tự tạo ra các đối tượng (thông qua từ khoá new)
+ Tìm kiếm và kết nối các đối tuộng phụ thuộc 
+ Quản lý vòng đời (life cycle) của chúng
- Sử dụng Ioc: Spring Framework chịu trách nhiệm kiểm soát việc khởi tạo, quản lý và kết nối các đối tượng. Chỉ cần khai báo các đối tượng cần thiết. 

=> Với Ioc, quyền điều khiển (control) việc quản lý các đối tượng bị đảo ngược từ lập trình viên sang Framework.


4. Spring Bean (Bean)
ĐỊNH NGHĨA:
- Spring Bean là một đối tượng Java thông thường (Plain Old Java Object - POJO) được quản lý với Spring Ioc Container, được cung cấp khi cần thiết.
- Hiểu ngắn gọn: Bean là những Object thuộc về Spring, không thuộc về bạn.

PHÂN BIỆT:
- Không phải tất cả các đối tượng đều là Bean hết:
+ Một đối tượng chỉ trở thành Bean thì nó được Container quản lý
+ Nếu bạn tự tạo đối tượng với keyword new trong code, thì đối tượng đó không phải là Spring Bean và sẽ không được thừa hưởng các tính năng của Spring (DI, AOP, ...)


5. Spring Context (Application Context)
- Nếu Bean là các đối tượng, thì Context chính là môi trường chứa và quản lý các đối tượng đó.

ĐỊNH NGHĨA:
- Spring Contenxt chính là giao diện chính để tương tác với IoC Container. Trong các ứng dụng Spring hiện đại (bao gồm cả Spring Boot) thì giao diện được sử dụng phổ biến nhất là ApplicationContext.

CHỨC NĂNG:
- ApplicationContext chịu trách nhiệm cho:
+ Tải (Load) các định nghĩa Bean (từ cấu hình)
+ Khởi tạo và quản lý toàn bộ Bean trong ứng dụng
+ Cung cấp cơ chế tìm kiếm và truy cập các Bean 

MỐI QUAN HỆ:
- Container (ApplicationContext) là nơi chứa tất cả các Bean và quản lý vòng đời của chúng. 
Cần phải có một Context đang chạy để có thể sử dụng bất kì Bean nào.
